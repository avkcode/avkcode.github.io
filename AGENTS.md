# Repository Instructions

- This repository is a static GitHub Pages site. Publish changes by committing the edited site files and pushing to `origin/main`.
- Keep publishes narrow. When the worktree contains unrelated edits or experiments, stage and commit only the files required for the requested change.
- There is no in-repo build step to update the site. Prefer direct edits to the shipped HTML, CSS, and asset files already served from the repository root.
- For blog posts that embed Mermaid diagrams inside dark cards or dark sections, use a dark-compatible Mermaid palette so node labels, borders, and connectors remain readable.
- Before publishing content changes, verify the affected page locally with a simple HTTP server and browser rendering when practical.
